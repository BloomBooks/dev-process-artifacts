What happened

This is a made-up demo to prove the public-hosting path end to end — none of the numbers below are real.

A NullReferenceException fires in ThumbnailGenerator.MakeThumbnail() when a book's cover-image path is null, crashing the thumbnailer during collection load.

Root cause

Legacy books can carry an empty <img> src. MakeThumbnail() assumes a non-null path and dereferences it without a guard.

Over to you

1. How should a null cover path be handled?

When the cover path is null we can either skip thumbnail generation for that book (leaving the default placeholder) or generate a generic "no cover" thumbnail so the slot isn't blank.

2. Screen out the noisy disposal exception?

A second, unrelated ObjectDisposedException from the same view fires ~40×/day only during shutdown and is harmless. We can add a tightly-scoped Sentry BeforeSend filter for it.
